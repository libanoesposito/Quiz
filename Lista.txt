Script

•	firebaseConfig → Configurazione Firebase.
	•	firebase.initializeApp → Inizializza Firebase.
	•	db → Firestore database.
	•	dbUsers → Database utenti locale (localStorage).
	•	window.alert → Override alert nativo con popup stile Apple.
	•	state → Stato globale dell’app (utente, progressi, modalità).
	•	session → Variabile sessione attiva (vuota).
	•	globalClassifica → Array dati classifica globale.
	•	ADMIN_PIN / TESTER_PIN → PIN speciali admin e tester.
	•	testerUser → Profilo utente tester di default.
	•	window.onload → Funzione iniziale all’apertura pagina.
	•	initTheme() → Imposta tema da localStorage o preferenza OS.
	•	toggleTheme() → Cambia tema light/dark e icona.
	•	updateNav(showBack, backTarget) → Aggiorna header (indietro/esci).
	•	saveMasterDB() → Salva progressi su Firebase e localStorage.
	•	updateGlobalLeaderboard() → Aggiorna classifica globale sul cloud.
	•	syncToFirebase(pin, userData) → Sincronizza singolo utente su Firebase.
	•	renderLogin() → Mostra schermata login iniziale.
	•	uiPin(type) → Mostra input PIN per login o registrazione.
	•	isWeakPin(pin) → Controlla se un PIN è troppo semplice.
	•	registerUser() → Registra nuovo utente su Firebase e locale.
	•	validatePin(type) → Controlla PIN esistente e gestisce login/registrazione.
	•	setGuest() → Imposta sessione guest.
	•	showHome() → Mostra schermata principale con percorsi, profilo, ripasso, classifica, pannello admin.
	•	domandaRepo → Repository domande per livelli (usata in showHome).

•	showLevels(lang) → Mostra i livelli di un linguaggio con progressi e blocchi.
	•	startStep(lang, lvl) → Inizia un livello, carica domande o sfide L5.
	•	updateEditor(text) → Aggiorna il contenuto dell’editor con evidenziazione sintassi Prism.
	•	syncScroll(el) → Sincronizza lo scroll tra textarea e display evidenziato.
	•	handleTab(e, el) → Inserisce 4 spazi alla pressione del tab in editor.
	•	renderL5(lang, index = null) → Mostra sfide livello 5 (esame).
	•	checkL5(lang, index) → Controlla il codice scritto dall’utente e aggiorna progressi.
	•	handleInput(el, lang) → Evidenzia il testo in un contenteditable e riposiziona il cursore.
	•	handleSpecialKeys(e) → Gestisce tasti speciali in contenteditable (Tab).
	•	renderQ() → Mostra la domanda attuale con opzioni e barra di progresso.
	•	markNotStudied(idx) → Segna una domanda come “non studiata” e aggiorna lo storico.
	•	check(isOk, userAnsText) → Valida risposta scelta, aggiorna ripasso, storico e progressi.
	•	next() → Passa alla prossima domanda o completa il livello aggiornando progressi.
	•	logout() → Termina sessione, resetta stato e torna al login.
	•	ensureUserId() → Garantisce che l’utente abbia un ID unico, lo assegna se mancante.
	•	calcStats() → Calcola statistiche generali dell’utente (totali, giuste, sbagliate, %).
	•	toggleSecurity(el) → Mostra/nasconde contenuto di sicurezza (profilo).

•	renderProfile() → Mostra il profilo utente con statistiche, progressi e storico.
	•	toggleGeneralProgress(card) → Mostra/nasconde i dettagli dei progressi generali.
	•	toggleGeneralContent(id, card) → Mostra/nasconde sezioni come sicurezza o storico.
	•	toggleHistory(el) → Mostra/nasconde uno specifico storico utente.
	•	generateHistoryHTML(u) → Genera l’HTML dello storico per un utente.
	•	toggleLangDetails(el) → Mostra/nasconde dettagli di un linguaggio nello storico.
	•	toggleCard(el) → Mostra/nasconde il contenuto di una card (profilo, sicurezza, ripasso).
	•	renderRipasso() → Mostra domande da ripassare (sbagliate o non studiate) con spiegazioni.
	•	resetStats() → Azzera progressi e storico utente, aggiornando DB e profilo.
	•	deleteAccount() → Segna l’account come eliminato e chiama logout.
	•	GUEST_LIMITS → Limiti per modalità guest per livello.
	•	guestLimitReached(lvl, idx) → Controlla se un guest ha raggiunto il limite domande.
	•	startStep(lang, lvl) → Override soft per guest; limita livelli e numero domande.
	•	renderQ() → Override soft per guest; mostra la domanda corrente con limiti demo.
	•	next() → Override soft per guest; passa alla domanda successiva.
	•	showGuestEndModal() → Mostra messaggio fine demo guest.
	•	showGuestLocked() → Mostra messaggio blocco livelli avanzati guest.
	•	renderAdminPanel() → Mostra pannello admin con utenti attivi, eliminati e azioni globali.
	•	togglePinVisibility(id) → Mostra/nasconde PIN utente in admin panel (richiamata in HTML).
	•	showUserHistory(id) → Mostra storico utente specifico (richiamata in HTML).
	•	adminDeleteUser(id) → Elimina un utente dal cloud (soft delete).
	•	adminRestoreUser(id, docId) → Ripristina un utente eliminato.
	•	adminPermanentDelete(docId) → Elimina definitivamente un utente dal cloud.
	•	adminClearTrash() → Svuota tutti gli utenti eliminati.
	•	adminResetAll(type) → Reset globale dei dati utenti (FULL o STATS).
